# Design: Issues #32, #33, #34

## Issue #33: Group extraction in batch mode

Add `--group`/`-g` flag to `--print` mode. Accepts a number (capture index) or string (named group).

**Changes:**
- `cli.rs`: Add `--group`/`-g` arg (`Option<String>`, requires `--print`)
- `app.rs`: Modify `print_output()` to accept optional group specifier, filter captures
- `main.rs`: Pass group arg to `print_output()`

**Behavior:**
- `rgx -p -g 1 'pattern' --text "input"` → print capture group 1
- `rgx -p -g name 'pattern' --text "input"` → print named capture group
- Missing group → stderr error, skip match, exit code 1
- No `-g` → existing behavior (full match)

## Issue #32: Performance comparison mode

`Ctrl+B` opens overlay showing compile + match time for each engine.

**Changes:**
- `app.rs`: Add `show_benchmark: bool`, `benchmark_results: Vec<BenchmarkResult>`
- `app.rs`: Add `run_benchmark()` method — iterates `EngineKind::all()`, compiles + matches, records timing
- `input/mod.rs`: Add `Action::Benchmark`, map `Ctrl+B`
- `ui/mod.rs`: Add `render_benchmark_overlay()` — table with engine name, compile time, match time, total, highlight fastest
- `main.rs`: Handle overlay navigation (Esc to dismiss)

**BenchmarkResult struct:**
```rust
pub struct BenchmarkResult {
    pub engine: EngineKind,
    pub compile_time: Duration,
    pub match_time: Duration,
    pub match_count: usize,
    pub error: Option<String>,
}
```

## Issue #34: Neovim plugin

Lua plugin at `plugin/nvim/` with `:Rgx` command in floating terminal.

**Files:**
- `plugin/nvim/lua/rgx/init.lua` — setup(), open(), config
- `plugin/nvim/README.md` — installation + usage docs

**Features:**
- `:Rgx` — open rgx in floating terminal
- `:Rgx pattern` — pre-fill pattern
- Visual select + `:Rgx` — use selection as `--text`
- Configurable window size (default 80% of editor)
